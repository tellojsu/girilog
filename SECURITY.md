# Security Policy

## Reporting Security Vulnerabilities

We take security seriously. If you discover a security vulnerability in girilog, please **do not** open a public issue on GitHub. Instead, please report it responsibly by emailing us at **josue.d.tello@gmail.com** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

We will acknowledge your report within 48 hours and work with you to address the issue before any public disclosure.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Supported       |
| Older   | ⚠️ Case by case    |

## Security Best Practices

When using girilog:

1. Keep your dependencies up to date
2. Review code before merging
3. Use environment variables for sensitive data
4. Enable GitHub's security features (Dependabot, code scanning)
